# Market Lens — Implementation Plan

## Overview

**Market Lens** is a Claude Code plugin that audits any product/project through Kotler's marketing frameworks. It acts as a provocative sparring partner — challenging builders to think beyond features and consider their product from the customer's perspective.

**Positioning:** Builder's product-thinking toolkit — not a "marketing tool for devs", but an essential skill for the next generation of engineers who ship entire products, not just code.

## Plugin Identity

- **Name:** `market-lens`
- **Commands:** `/market-lens:audit` (full), `/market-lens:quick` (myopia only)
- **Auto-skill:** `product-thinking` (triggers when Claude detects product work)
- **Tone:** Provocative sparring partner — direct, challenging, constructive

## The 4 Lenses (Kotler-based)

### Lens 1: Myopia Check
**Question:** "Are you selling the drill or the hole?"
- Detects feature-obsession vs problem-solving
- Scans: README, landing page copy, feature naming, navigation
- Looks for: technical jargon vs customer language

### Lens 2: Value Perception
**Question:** "What does the customer actually get vs what they pay?"
- Maps perceived value (benefits) against costs (money, time, effort, psychological)
- Scans: pricing page, onboarding flow, signup friction, free vs paid features
- Looks for: value communication gaps

### Lens 3: Satisfaction Risk
**Question:** "Where will expectations break?"
- Compares marketing promises vs actual delivery
- Scans: marketing copy vs real UI, error messages, loading states, empty states
- Looks for: over-promise / under-deliver points
- Motorola test: "If the customer doesn't like it, it's a defect"

### Lens 4: Retention Signal
**Question:** "Built for one-time use or long-term relationships?"
- Checks for retention mechanisms and relationship-building
- Scans: engagement hooks, email/notification code, returning user flows
- Looks for: presence/absence of retention patterns
- 5x rule: acquiring new customers costs 5x more than retaining existing ones

## Flow

### `/market-lens:audit` (Full Mode)

```
1. AUTO-SCAN
   └── Silently scan codebase (README, package.json, UI copy, etc.)

2. CONTEXT QUESTIONS (3 max)
   ├── Q1: "What problem does your user have BEFORE they find your product?"
   ├── Q2: "Who specifically uses this — and why them?"
   └── Q3: "What would they use if your product didn't exist?"

3. LENS-BY-LENS CHALLENGES (interactive)
   ├── Lens 1: Myopia Check
   │   ├── Present findings from scan
   │   ├── Challenge the user
   │   ├── Discuss & resolve
   │   └── Record finding + risk level
   ├── Lens 2: Value Perception
   │   └── (same pattern)
   ├── Lens 3: Satisfaction Risk
   │   └── (same pattern)
   └── Lens 4: Retention Signal
       └── (same pattern)

4. FINAL REPORT
   ├── Summary of all findings
   ├── Risk levels (green/yellow/red per lens)
   ├── Top 3 Action Items (highest impact)
   └── Save to PRODUCT-AUDIT.md
```

### `/market-lens:quick` (Quick Mode)

```
1. AUTO-SCAN (same)
2. Q1 only: "What problem does your user have BEFORE they find your product?"
3. Myopia Check only (challenge format)
4. Quick finding + 1 action item
```

### `product-thinking` (Auto-Skill)

Triggers automatically when Claude detects the user is working on:
- README or landing page copy
- Pricing pages
- Onboarding flows
- Feature descriptions

Provides subtle nudges like: "You're describing what this does, but not why someone would care."
